# Additional PWA Requirements

## UI Fix: Session Card Heart Icon

**IMPORTANT**: The heart/favorite icon should be positioned in the **bottom-right corner** of the session card, NOT as a full-width area at the bottom.

Current (wrong):
```
┌─────────────────────────┐
│ 09:00 - 09:45  Room     │
│ Session Title    [tag]  │
│ Speaker Name            │
│ ● Track                 │
│                         │
│      ❤️ Saved           │  ← Wrong: full-width centered area
└─────────────────────────┘
```

Expected (correct):
```
┌─────────────────────────┐
│ 09:00 - 09:45  Room     │
│ Session Title    [tag]  │
│ Speaker Name            │
│ ● Track            ❤️   │  ← Correct: small icon bottom-right
└─────────────────────────┘
```

The heart should be a small clickable icon, not a large button with text.

## Feature: My Tickets

Add a "My Tickets" section to the attendee PWA (`/app/[editionSlug]`) that allows users to:

### Core Features
1. **View their tickets** - Display all tickets associated with the user's email
2. **Show ticket details**:
   - Ticket type name
   - Attendee name
   - QR code for check-in
   - Ticket status (valid, used, cancelled)
   - Order information
3. **Download ticket as PDF** - Use the existing `pdf-ticket-service.ts`

### Implementation hints
- Use the existing `billing_tickets` collection
- Look at how `/tickets/[editionSlug]/confirmation/+page.svelte` displays tickets
- The ticket lookup can be done by email (from the order) or by a ticket access token
- Consider adding a "Find my tickets" form where users enter their email and receive a link
- Reuse the existing `QRCodeDisplay` component for showing QR codes
- Use the existing `generateTicketPdf` function from `pdf-ticket-service.ts`

### User flow (Option B - Direct ticket URL recommended for mobile)
1. Tickets already have a unique URL/token
2. User bookmarks or saves the confirmation page URL
3. PWA stores ticket access in localStorage for quick access
4. Show "My Tickets" tab if user has saved tickets

### Routes to add
- `/app/[editionSlug]/tickets` - List of user's tickets
- `/app/[editionSlug]/tickets/[ticketId]` - Single ticket view with QR code
- `/app/[editionSlug]/tickets/find` - Email lookup form (optional)
